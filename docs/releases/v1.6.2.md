# PnP Bridge 1.6.2

A Day-0 claim fix for the management-interface mask.

## Fixed

- **Management subnet is now sent to Catalyst Center as a full dotted mask.**
  CCC onboarding templates apply `mgmt_subnet` as the interface mask
  (`ip address <ip> <mask>`), and IOS rejects CIDR/prefix form there — a real
  claim failed with `PnP Service Error 1413. Invalid input detected` on
  `ip address 172.20.10.249 172.20.10.0/24`. The Day-0 preview still shows the
  CIDR (`172.20.10.0/24`) so it stays readable, but the value sent to CCC is now
  the dotted mask (`255.255.255.0`). An operator override for that field still
  takes precedence.

## Upgrade notes

- No schema change. `git pull && make image`, recreate the container.
