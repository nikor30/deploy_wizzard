# PnP Bridge 1.6.0

Day-0 onboarding now fills the real Catalyst Center template variables from
NetBox, so a claim needs (almost) no manual entry — and global variables cover
the rest.

## New

- **More Day-0 values derived from NetBox.** In addition to hostname / mgmt IP
  / mask / prefix / VLAN, the Day-0 step now prefills:
  - `mgmt_subnet` — from the device's management IP
  - `mgmt_vlan_name` — the name of the VLAN you picked in the match step
  - `switchType` / `campusswitch` / role — the NetBox device **role** (e.g.
    `access`)
  - `site` / `location` / `rack` — from the device
  For the real `00_Webasto_OnBoarding` template this means only the AES key
  stays open. `mgmt_subnet` format (network/prefix) and the role-based fields
  should be confirmed against your template with the debug switch on.
- **Global variables (set once, used everywhere).** A stored secret now
  **auto-fills any Day-0 or Day-N template variable with the same name** — name
  one `AES_ENCRYPTION_KEY` and it fills that variable on every device and every
  template, no explicit `secret.<name>` mapping needed. Values stay encrypted
  and masked, decrypted only for the claim/deploy call. (Settings → Day-N
  Variables → **Global variables & secrets**.)

## Upgrade notes

- No schema change. `git pull && make image`, recreate the container.
- Turn on Settings → Credentials → **Debug** and run a Day-0 prepare to see the
  source of every variable and confirm the derivations against your template.
