# PnP Bridge 1.1.0

The onboarding wizard is complete: this release adds phases **P3–P5** of the roadmap —
the full Day-0 → Day-N → finalize flow on top of the 1.0.0 foundation.

## New

- **Onboarding wizard, steps 1–5**
  - **Select devices:** live table of unclaimed Catalyst Center PnP devices with
    multi-select, serial/PID filter and 60 s auto-refresh.
  - **Match with NetBox:** serial-based matching against `planned` devices (normalized
    on both sides), mgmt IP from `primary_ip4` with mgmt-interface fallback, site
    resolution via the mapping table, per-site mgmt-VLAN dropdown. Unmatched or
    unmapped devices can never be claimed.
  - **Day-0 claim:** template + optional image, variables prefilled from NetBox,
    per-device isolated claiming with live progress (SSE), PnP state polling.
  - **Day-N provision:** template variables auto-filled via configurable NetBox
    dot-path mappings (Settings → Day-N Variables); anything unresolvable becomes a
    required manual input. Task polling drills the CCC task tree for buried errors.
  - **Finalize:** devices are set `active` in NetBox **only** after verified Day-N
    success; activation failures yield a `partial_success` job. Summary screen with
    per-device outcomes.
- **Resumable jobs:** all wizard state lives server-side — close the browser at any
  step and resume later.
- **ISE webhook:** signed (`X-PnPB-Signature`, HMAC-SHA256) `day0_success`
  notifications with retries; delivery outcomes stored for the upcoming Logs page.
- **Zero-config start:** the container now starts without any environment variables;
  the encryption key is generated on first start and persisted on the `/data` volume.

## Notes

- Cisco/NetBox payload shapes follow the specification baseline; capture live
  CCC 2.3.7 fixtures before first production use (see `PLAN.md`).
- Statistics dashboard and the searchable Logs page arrive in the next phase (P6).
