# PnP Bridge 1.6.1

Two Day-0 variable fixes from live use of the onboarding preview.

## Fixed

- **`campusswitch` is now a yes/no picker.** It was wrongly derived from the
  NetBox device role (showing e.g. `access`). It is an operator decision, so the
  Day-0 preview now renders it as a **no/yes dropdown** (default `no`). The
  selected value is sent in the claim. `switchType` still uses the NetBox role,
  which is correct.
- **Catalyst Center's garbled password variables are hidden.** CCC has a bug
  that leaks random secret tokens into a template's parameter list — names like
  `pPYzdaRZdKO5gppL7ddKhk3iF` or `OaMGKyQBNwDjxFcagpT`. These opaque mixed-case
  tokens (no separators, long, with digits or many case flips) are now detected
  and dropped entirely: they never appear in the Day-0 preview and are never
  sent in the claim.

## Upgrade notes

- No schema change. `git pull && make image`, recreate the container.
