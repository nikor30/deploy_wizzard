# PnP Bridge 1.6.3

A Day-0 fix so onboarded devices get their NetBox name.

## Fixed

- **The device hostname is now set during the claim.** A switch onboarded via
  Day-0 but kept the default IOS hostname `Switch` — the NetBox name never
  reached it. The `00_Webasto_OnBoarding` template has no hostname variable; its
  `SET_HOSTNAME` task reads Catalyst Center's device record (the "Device Name"
  in the PnP claim UI), which the tool never populated. The site-claim now
  carries a top-level `hostname` set to the NetBox device name, so the box comes
  up correctly named. When a matched device has no NetBox name, the field is
  omitted and CCC's behaviour is unchanged.

## Upgrade notes

- No schema change. `git pull && make image`, recreate the container.
- Only affects new claims; already-onboarded devices keep whatever hostname they
  have until re-provisioned.
