# PnP Bridge 1.5.0

The Day-0 claim step now previews the selected template's variables — so you
can see what is prefilled from NetBox and fill anything still open (like the
gateway) before claiming — plus a global debug switch.

## New

- **Day-0 template variable preview.** Selecting an onboarding template
  introspects its variables and resolves each per device:
  - **Prefilled** from the NetBox match — HOSTNAME, MGMT_IP, MGMT_MASK,
    MGMT_PREFIX, MGMT_VLAN (shown read-only), and anything mapped via a Day-N
    dot-path.
  - **Open** fields you fill inline — including the **gateway**, pre-filled
    with a best-effort guess (the management subnet's first host) that you
    confirm or override.
  Empty open fields are simply omitted from the claim.
- **Gateway.** Previously missing; it now appears as an editable field with a
  sensible default.
- **Global debug switch** (Settings → Credentials → Debug): when on, the Day-0
  and Day-N steps show the **source** of every variable (netbox / mapped /
  manual), so you can verify exactly what the tool prefilled and what is still
  open before deploying.

## Upgrade notes

- Migration `0009` (Day-0 variables + a small settings table) runs
  automatically at startup. No action required.
