# PnP Bridge 1.3.2

Bugfix release.

## Fixed

- **Failed/reset switches were invisible in the wizard.** The device-selection
  step listed only Catalyst Center PnP devices in state `Unclaimed`. A switch
  that failed a previous onboarding attempt lingers in CCC in state `Error`,
  `Planned`, or `Onboarding` — CCC keeps the old PnP record even after the
  switch is factory-reset — so it showed in CCC's own GUI but dropped out of
  the wizard entirely. The wizard now lists devices across all actionable PnP
  states (`Unclaimed`, `Planned`, `Onboarding`, `Error`), each shown with a
  colored state badge, so they can be picked and (re-)claimed. A hint explains
  that non-`Unclaimed` rows are earlier attempts; if a stale record blocks a
  re-claim, delete it in Catalyst Center first.

## Upgrade notes

- `git pull && make image`, then recreate the container. No schema changes.
