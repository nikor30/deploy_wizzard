# PnP Bridge 1.3.1

Bugfix release.

## Fixed

- **No web GUI after upgrading from ≤ 1.1** with an existing `/data` volume:
  1.2.0 switched the container to a non-root user (uid 10001), but volumes
  created by the older root-running containers stayed root-owned — the app
  could not open its database/key and crashed at startup. The container
  entrypoint now starts as root only to repair `/data` ownership and
  immediately drops privileges to uid 10001. Running with `--user 10001` on
  an already-writable volume keeps working unchanged.
- CI now includes a regression test that boots the image against a
  root-owned volume from a simulated pre-1.2 container and verifies both the
  health endpoint and that the app process runs as the non-root user.

## Upgrade notes

- `git pull && make image`, then recreate the container — no manual
  `chown` needed; the entrypoint fixes the volume on first start.
