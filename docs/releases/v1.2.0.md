# PnP Bridge 1.2.0

Wizard usability fixes from live testing, statistics & logging (P6), and the
full hardening phase (P7): mock-server test stacks, an end-to-end browser
suite, a hardened container, and operations docs.

## New

- **Statistics dashboard (`/stats`):** claimed/provisioned/failed totals with
  time filters, success rate, average Day-0/Day-N durations, failures by error
  category, devices-per-day chart.
- **Logs page (`/logs`):** searchable, filterable (job, serial, level,
  component, time range) view of the structured log sink with expandable
  redacted request/response context and a **retry** action for failed ISE
  webhook deliveries. Retention defaults to 90 days (nightly cleanup).
- **Wizard usability** (from live-testing feedback):
  - Back navigation: jobs ← matching ← Day-0.
  - **Re-run matching** button — fix a device in NetBox and retry without
    starting a new job.
  - Info banner explaining the match requirements (same serial number,
    NetBox status `planned`).
  - Jobs can be **deleted** from the start view (running jobs are protected
    until they finish).
- **Mobile layout:** the sidebar collapses to a top navigation bar on small
  screens; tables scroll inside their own containers.
- **Mock CCC/NetBox/ISE stack** (`tests/mocks/`) with failure injection —
  also runnable standalone for demos:
  `uv run python -m tests.mocks.stack --port 9100`.
- **Docs:** `docs/runbook.md` (deploy, upgrade, backup, troubleshooting
  matrix) and `SECURITY.md`.

## Fixed (live Catalyst Center)

- Wizard 500 when CCC returns the PnP list as a bare JSON array.
- First unclaimed device missing from the list (PnP paging is 0-based).
- Empty *Source IP* column (IP read from `httpHeaders`/`ipInterfaces`).
- Unhandled errors now return JSON details and appear on the Logs page.

## Hardening

- **Container:** runs as a dedicated non-root user (uid 10001), ships a
  `HEALTHCHECK` on `/api/health`, minimal build context.
- **Integration suite:** real clients against the mock stack over HTTP —
  full wizard happy path, half-failed batch isolation, task-tree error
  drilling, webhook failure without claim rollback, NetBox PATCH failure ⇒
  `partial_success`, 5xx retry, and a 25-device load test proving the
  5-request CCC rate limit.
- **Playwright e2e suite** (`make e2e`, own CI job): complete wizard run with
  resume-after-reload and Day-N manual entry, half-failed batch, settings
  round-trip with masked secrets, mobile smoke.

## Upgrade notes

- No schema changes requiring action — migrations run automatically at start.
- The container now runs as uid 10001: named volumes keep working unchanged;
  **bind mounts** must be writable for that uid (`chown 10001 <dir>`, or
  podman `--userns=keep-id`; SELinux hosts need `:Z`). See `docs/runbook.md`.
