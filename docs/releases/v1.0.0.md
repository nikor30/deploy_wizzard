# PnP Bridge 1.0.0

First tagged release. Covers implementation phases **P0–P2** of the roadmap (see
`PLAN.md`).

## Included

- **Application skeleton**: FastAPI backend serving the built React SPA on port **8060**,
  health endpoint (`/api/health`), structured JSON logging with secret redaction,
  SQLite + Alembic migrations applied automatically at startup.
- **Encrypted credential settings** for Catalyst Center, NetBox and the ISE webhook:
  secrets Fernet-encrypted at rest (`PNPB_SECRET_KEY` required), write-only API with
  masked reads (`****abcd`), per-service **Test connection** buttons.
- **API clients**: Catalyst Center (token auth with 401-refresh-once + proactive 55-min
  refresh, 5-connection rate limit, pagination) and NetBox (token auth, next-link
  pagination) — both with timeouts, retries and typed, user-actionable errors.
- **Site mapping**: two-column NetBox ↔ Catalyst Center mapper with search, unmapped
  highlighting, JSON export/import; persisted server-side.
- **Container**: single OCI image (Podman/Docker), multi-stage build, runs as
  `uvicorn` on `:8060` with a `/data` volume.

## Not yet included (upcoming)

- Onboarding wizard steps 1–5 (device selection, NetBox matching, Day-0 claim,
  Day-N provisioning, finalize) — phases P3–P5.
- Statistics dashboard and searchable log UI — phase P6.

## Running

```bash
podman run --rm -p 8060:8060 \
  -e PNPB_SECRET_KEY=<fernet-key> \
  -v pnpb-data:/data \
  pnp-bridge:1.0.0
```

Generate a key with:
`python -c 'from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())'`
